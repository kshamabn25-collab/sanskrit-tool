Troubleshooting

Common setup issues on macOS and Windows — with step-by-step fixes.

macOS

"Script Tool is damaged and can't be opened" or "cannot be verified"

macOS Gatekeeper blocks apps that aren't signed with an Apple Developer certificate. Script Tool is safe — this is a false positive. Use one of the methods below to open it.

Method 1 — System Settings (recommended)

  1. When the alert appears, click Cancel (don't click Move to Trash).
  2. Open Apple menu → System Settings.
  3. Click Privacy & Security in the sidebar.
  4. Scroll down — you'll see "Script Tool was blocked from use because it is not from an identified developer."
  5. Click Open Anyway, then confirm with Open in the dialog that appears.
  6. Script Tool will open normally from now on.

Method 2 — Right-click trick

  1. In Finder, right-click (or Ctrl-click) on Script Tool.app.
  2. Select Open from the context menu.
  3. A new dialog appears with an Open button — click it.
  4. Script Tool opens and is now whitelisted permanently.

Method 3 — Terminal (one command)

  1. Press Cmd+Space, type Terminal, press Return.
  2. Paste and run:
xattr -cr "/Applications/Script Tool.app"

This removes the quarantine flag that triggers the Gatekeeper warning.

macOS

Sanskrit IME is ON but nothing types / keystrokes aren't intercepted

The global Sanskrit IME requires Accessibility permission to intercept keystrokes system-wide. Without it, pressing Alt+Shift+M activates the HUD bar but keys still go to the app as normal text.

  1. Open Apple menu → System Settings.
  2. Click Privacy & Security in the sidebar.
  3. Click Accessibility.
  4. Find Script Tool in the list.
  5. Toggle the switch to ON (blue).
  6. You may be prompted for your Mac password — enter it.
  7. Quit and relaunch Script Tool, then try Alt+Shift+M again.

If Script Tool doesn't appear in the list, try toggling the IME once (Alt+Shift+M) — this triggers the permission prompt which adds it to the list.

Windows

Windows SmartScreen says "Windows protected your PC"

Windows Defender SmartScreen warns about installers that don't have a paid Extended Validation (EV) certificate. Script Tool is safe. Follow the steps below.

  1. Double-click the downloaded Sanskrit-Tool-windows.exe.
  2. When the blue SmartScreen dialog appears, click More info (bottom-left text link).
  3. A Run anyway button appears — click it.
  4. The installer proceeds normally.

If you see a User Account Control (UAC) prompt asking "Do you want to allow this app to make changes?" — click Yes. This is normal for any installer.

macOS & Windows

Global shortcuts (Alt+Shift+I, Alt+Shift+M…) don't respond

Global shortcuts can be blocked by another app that already registered the same combination, or by the OS itself.

Check Script Tool is running

  1. macOS: look for the ॐ icon in the menu bar (top-right of your screen).
  2. Windows: look for the ॐ icon in the system tray (bottom-right, expand hidden icons with the ^ arrow).
  3. If the icon isn't there, Script Tool isn't running — open it from Applications (macOS) or the Start Menu (Windows).

Check for conflicting apps

  1. Another app (window managers, Logitech Options, Raycast, AltTab, etc.) may have claimed Alt+Shift+I or Alt+Shift+S.
  2. Quit the suspected conflicting app and try the shortcuts again.
  3. On Windows: Alt+Shift alone switches the keyboard input language if you have multiple languages installed. This shouldn't block Alt+Shift+I combos, but if it does — go to Settings → Time & Language → Language & Region and remove extra keyboard layouts, or change the language-switch hotkey to something else.

Restart Script Tool

  1. Click the ॐ tray icon → Quit.
  2. Reopen Script Tool.
  3. Try the shortcut again.
