Troubleshooting

Common setup blockers on macOS and Windows — with exact steps to fix them.

macOS

Getting Script Tool running on macOS

macOS applies three layers of security to unsigned apps. You need to clear all three before the inline IME works. Do these steps in order.

1

Allow the app to open — bypass Gatekeeper

macOS blocks apps not signed by a registered Apple developer. When you double-click Script Tool and see "cannot be verified" or "damaged":

  1. Click Cancel on the alert (don't move to Trash).
  2. Open Apple menu → System Settings → Privacy & Security.
  3. Scroll down — you'll see "Script Tool was blocked from use…"
  4. Click Open Anyway, then confirm with Open.

Alternatively: right-click Script Tool.app in Finder → Open → click Open in the dialog.

2

Clear the quarantine flag in Terminal

Even after allowing the app, macOS stamps a quarantine attribute that can cause issues later. Remove it permanently with one command:

  1. Press Cmd+Space, type Terminal, press Return.
  2. Paste and run the command below, then press Return.
xattr -cr "/Applications/Script Tool.app"

You only need to do this once. It removes all quarantine and extended attributes from the app bundle.

3

Grant Accessibility permission

The inline IME needs Accessibility access to intercept keystrokes system-wide. Without this, pressing Alt+Shift+M will appear to do nothing.

  1. Launch Script Tool — the ॐ icon appears in your menu bar.
  2. Go to System Settings → Privacy & Security → Accessibility.
  3. Find Script Tool in the list and toggle it ON.
  4. Enter your Mac password when prompted.
  5. Quit and relaunch Script Tool.
  6. Press Alt+Shift+M — the IME indicator should now appear at the top of your screen.

If Script Tool doesn't appear in the Accessibility list, toggle the IME once (Alt+Shift+M) — this triggers the permission dialog and adds it to the list.

Windows

Getting Script Tool running on Windows

Windows applies two security checks to downloaded executables — one in your browser and one when you run the file. Both need to be cleared.

1

Allow the download in your browser

Chrome and Edge may warn that the .exe is uncommon. This is expected for new unsigned apps.

Chrome

  1. After downloading, find the file in the bottom download bar or the Downloads page.
  2. Chrome may show "This file isn't commonly downloaded."
  3. Click the arrow (›) or the three-dot menu next to the file.
  4. Click Keep.

Microsoft Edge

  1. Edge shows a warning in the download sidebar or bar.
  2. Click the three-dot menu (…) on the download entry.
  3. Click Keep anyway.
2

Bypass Windows SmartScreen when running

When you double-click the installer, Windows Defender SmartScreen may show a blue warning screen. Click through it:

  1. Click More info (small text link, bottom-left of the blue screen).
  2. Click Run anyway.
  3. If a UAC prompt appears asking "Do you want to allow this app to make changes?" — click Yes.
  4. The installer runs normally.
macOS & Windows

Global shortcuts don't respond

1. Check Script Tool is running

  • macOS: look for the ॐ icon in the menu bar (top-right of your screen).
  • Windows: look for the ॐ icon in the system tray (bottom-right, expand hidden icons with the ^ arrow).
  • If the icon isn't there, Script Tool isn't running — open it from Applications (macOS) or Start Menu (Windows).

2. Check for conflicting apps

  • Another app (window managers, Logitech Options, Raycast, AltTab, etc.) may have claimed the same shortcut.
  • Quit the suspected app and try the shortcut again.
  • Windows only: Alt+Shift alone switches keyboard input languages if you have multiple layouts installed. Go to Settings → Time & Language → Language & Region and remove extra keyboard layouts, or change the language-switch hotkey.

3. Restart Script Tool

  • Click the ॐ tray icon → Quit, then reopen Script Tool.
